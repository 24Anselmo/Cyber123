1. Visão Geral

Sistema multiplataforma para deteção de cyberbullying em texto, desenvolvido para o Instituto Superior Politécnico da Lunda-Sul (Saurimo, Angola). Combina uma interface web (Flask) e uma aplicação desktop (Tkinter) que partilham a mesma base de dados SQLite e o mesmo motor de deteção enriquecido.

1.1 Funcionalidades Principais

FuncionalidadeDescrição
Análise de texto em tempo realClassifica como Crítico / Ofensivo / Suspeito / Neutro
Dicionário alargado181+ palavras ofensivas + 40+ gírias regionais angolanas
Classificação por níveiscrítico, alto, médio, baixo, neutro
Alertas inteligentesThreshold ≥ 50% de confiança (Suspeito+)
Interface web e desktopMesma BD partilhada, mesma experiência
Servidor web embutidoDesktop inicia Flask automaticamente após login
AutenticaçãoLogin protegido em ambos os sistemas
Controlo de acessoPapéis: admin e moderador
Auto-saveToda análise é guardada automaticamente na BD
Exportação de relatóriosCSV e TXT

  • Desktop e Web partilham o mesmo ficheiro cyberbullying.db
  • Desktop inicia servidor web Flask automaticamente como thread daemon
  • Motor de deteção (app/detector.py) é o mesmo para ambas as interfaces
  • Toda a análise é auto-salva na BD antes de ser exibida

2.3 Autenticação e Sessão

  • Web: sessão Flask armazenada em cookie assinado (secret_key)
  • Desktop: login verificado localmente via SQLite; user_id, nome e papel passados ao construtor da app
  • Logout (web): session.clear() + redirect para /login
  • Logout (desktop): root.destroy() + main() (re-inicia ciclo de login)
  • Timeout: as sessões web não expiram automaticamente (apenas ao fechar o browser ou clicar Sair)

3. Instalação e Configuração

3.1 Requisitos

  • Windows 10/11 ou Linux (Ubuntu 22.04+)
  • Python 3.10 ou superior
  • pip (gestor de pacotes Python)

3.2 Instalação de Dependências

cd cyberbullying_detector
pip install -r requirements.txt

Conteúdo do requirements.txt:

flask>=3.0
flask-sqlalchemy>=3.0
flask-cors>=4.0

3.3 Execução

Modo Web (Servidor Flask Autónomo)

python run.py

Acessar: http://localhost:5000

O run.py verifica automaticamente se existe uma base de dados em offline/dist/data/cyberbullying.db (criada pelo EXE). Se existir, utiliza essa para manter consistência.

Modo Desktop (Tkinter + Servidor Web Embutido)

cd offline
python desktop_app.py

Ou clicar duas vezes em iniciar.bat. Após login, o servidor web é iniciado automaticamente em http://localhost:5000.

Executável Compilado

offline/dist/CyberbullyingDetector.exe

3.4 Configuração

Variável de AmbienteDescriçãoPadrão
CYBERBULLYING_DB_PATHCaminho completo para a BD SQLitedata/cyberbullying.db

Se a variável não for definida, o caminho é calculado automaticamente:

  • Script: diretório raiz do projeto + /data/cyberbullying.db
  • EXE: diretório do executável + /data/cyberbullying.db

6. Motor de Deteção

6.1 Arquitetura

O motor (app/detector.py) é um singleton baseado em regras (rule-based), partilhado entre web e desktop. Utiliza:

  1. Lista hardcoded (self.pa): 181 palavras e expressões ofensivas em português
  2. Dicionário local (data/local_dictionary.json): termos carregados dinamicamente
  3. Gírias da BD (girias_db): 40+ gírias regionais da Lunda-Sul

6.2 Sistema de Pontuação

FatorPontos
Base (confiança inicial)5.0
Por palavra/gíria crítica+45
Por palavra/gíria alta+25
Por palavra/gíria média+15
Por palavra/gíria baixa+8
Bónus crítico (se pelo menos 1 termo crítico detetado)+35
MAIÚSCULAS (>50% do texto em maiúsculas)+10
Sinais de exclamação/interrogação+2 cada (máx. +10)
Máximo100

6.3 Thresholds de Classificação

ConfiançaClassificaçãoAlerta
≥ 90%🔴 CríticoSim
≥ 70%🟠 OfensivoSim
≥ 50%🟡 SuspeitoSim
< 50%⚪ NeutroNão

6.4 Nível de Gravidade

O nivel_geral é determinado pelo nível mais alto entre todas as palavras e gírias detetadas:

critico > alto > medio > baixo > neutro

8.2 Controlo de Acesso

Web (JavaScript):

const papel = document.documentElement.dataset.papel || 'moderador';
if (papel !== 'admin') {
    document.querySelectorAll('[data-admin]')
        .forEach(el => el.style.display = 'none');
}

Web (Python decorators):

def admin_required(f):
    @wraps(f)
    def decorated(*args, **kwargs):
        if session.get('papel') != 'admin':
            return jsonify({'erro': 'Acesso negado'}), 403
        return f(*args, **kwargs)
    return decorated

if self.papel == 'admin':

8.3 Segurança

  • Offline-first: o sistema funciona 100% offline (localhost)
  • Passwords em plaintext: aceitável para âmbito académico 100% offline
  • Servidor web: bind a 0.0.0.0:5000 — acessível apenas na rede local
  • Chave secreta: hardcoded (SECRET_KEY) — adequado para ambiente controlado

10. Manutenção e Resolução de Problemas

10.1 Log de Diagnóstico

O sistema escreve para data/debug.log (ou offline/dist/data/debug.log no EXE):

  • Estado da BD antes/depois da migração
  • Tentativas de login (utilizador, senha, resultado)
  • Erros de conexão à BD
  • Diagnóstico automático ao iniciar

10.2 Problemas Comuns

ProblemaCausa ProvávelSolução
"Usuário ou senha inválidos"BD corrompida ou migração incompletaApagar data/cyberbullying.db e reiniciar
"Database is locked"Concorrência entre web e desktopAguardar ou reiniciar. WAL mode + timeout 10s
Servidor web não inicia no desktopPorta 5000 ocupadaFechar outras instâncias do Flask
EXE não abrePyInstaller sem hidden importsRecompilar com .spec atualizado
Login falha após logout (desktop)Thread do web server ainda ativaFechar completamente o EXE e reabrir

10.3 Reset Completo

Para reiniciar o sistema do zero:

rem Apagar base de dados
del data\cyberbullying.db
rem ou (se usar EXE)
del offline\dist\data\cyberbullying.db

rem Apagar log
del data\debug.log

rem Reiniciar
python run.py
rem ou
offline\dist\CyberbullyingDetector.exe

Na primeira execução, o sistema recria a BD e insere os dados iniciais.

10.4 Verificação de Integridade

No desktop, o diagnóstico automático corre ao iniciar (_diagnose_db()) e regista:

  • Caminho da BD
  • Se o ficheiro existe
  • Número de análises, comentários e alertas
  • Últimas 5 análises
